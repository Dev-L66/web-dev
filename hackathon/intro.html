Introduction to Web Development

What is Internet?

Internet is a global communication system that links together thousands of individual networks. It allows exchange of information between two or more computers on a network. Thus internet helps in transfer of messages through mail, chat, video & audio conference, etc.

What is an IP address?

An IP (Internet Protocol) address is a unique numerical label assigned to every device on a computer network that uses the Internet Protocol for communication.

  • It provides the network location of that device, enabling other devices to find and communicate with it.
  • IP addresses are crucial for sending and receiving information, ensuring data goes to the correct destination.
  • The IP address is assigned by the Internet Service Provider (ISP).

What is DNS?

DNS stands for the Domain Name System, a crucial Internet system that translates human-friendly domain names (like www.example.com) into machine-readable IP addresses (like 192.0.2.1). It acts as the "phonebook of the Internet," allowing users to access websites and other online resources by simply typing a name into their browser instead of a complex numerical IP address. DNS servers handle these translation requests, known as DNS lookups, enabling devices to find and communicate with each other across the network.

How DNS Works?

  • User Request: You type a domain name (e.g., example.com) into your web browser.
  • DNS Query: Your computer sends a query to a DNS server, asking for the IP address associated with that domain name.
  • DNS Resolution: The DNS server looks up the IP address in its records.
  • IP Address Found: If the IP address is found, the DNS server returns it to your computer.
  • Connection: Your browser then uses this IP address to connect to the website's server and load the page.

What is MAC address?

A MAC (Media Access Control) address is a unique, hardware-based identifier for a network interface controller (NIC) like an Ethernet port or Wi-Fi adapter, functioning as a device's permanent "name tag" on a local network segment to ensure data goes to the correct place.

What is port number?

Port numbers are 16-bit numerical identifiers (0-65535) that act as communication endpoints for specific processes or services on a networked device, working with an IP address to form a unique connection. They allow multiple applications to run on a single device and send or receive data over a network using protocols like TCP and UDP. Common port numbers include 80 for HTTP web traffic, 443 for HTTPS, and 22 for SSH.

IPV4 vs IPV6

IPv4 and IPv6 are different versions of the Internet Protocol, the system that assigns unique addresses to devices on the internet. IPv4 is the older, 32-bit version with a limited number of addresses, while IPv6 is the newer, 128-bit version with a vastly larger address space to accommodate the growing number of internet-connected devices. IPv6 is also superior in terms of built-in security features like IPsec, more efficient routing, and auto-configuration capabilities.

Web development

    When a client (typically a web browser) requests a web page from a server, the server responds by sending the necessary files to render that page. This process involves the following:
  • Client Request: The browser sends an HTTP request to the server, asking for a specific resource (e.g., an HTML file).

  • Server Response (HTML): The server sends an HTTP response back to the client, containing the requested HTML file.

  • Browser Rendering (HTML): The browser receives the HTML and starts parsing it, as it encounters references to other resources (like CSS files and JavaScript files) within the HTML.

  • Client Request (CSS): The browser sends a separate HTTP request to the server to fetch the linked CSS file.

  • Server Response (CSS): The server responds to these requests by sending the corresponding CSS files.

  • Browser Rendering (CSS): The browser applies the styles defined in the CSS file to the HTML elements, determining the visual presentation of the page (colors, fonts, layout, etc.).

  • Client Request (JavaScript): The browser sends a separate HTTP request to the server to fetch the linked JavaScript file.

  • Server Response (JavaScript): The server responds to these requests by sending the corresponding JavaScript files.

    • This sequence of requests and responses allows the client to receive all the necessary components (HTML for structure, CSS for styling, and JavaScript for interactivity) to fully render and display a functional web page.

MERN STACK

MERN (MongoDB, Express.js, React, Node.js) is a popular web development stack that consists of four main components: MongoDB, Express.js, React, and Node.js.

Here's a brief overview of each component:

  • MongoDB: MongoDB is a NoSQL database that stores data in a flexible, JSON-like format. It is known for its high scalability and easy-to-use interface.
  • Express.js: Express.js is a minimal and flexible web application framework for Node.js. It provides a set of tools and features for building web applications and APIs.
  • React: React is a JavaScript library for building user interfaces. It uses a component-based architecture, making it easy to manage complex UIs.
  • Node.js: Node.js is a runtime environment for executing JavaScript code on the server-side. It allows you to build server-side applications using JavaScript.

What is a fullstack developer?

A Full Stack Developer is someone who works with both the frontend (client side) and backend (server side) of web/mobile applications. They can design, build, deploy, and maintain the whole application from what users see to what runs behind the scenes.
